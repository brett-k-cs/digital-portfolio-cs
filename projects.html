Goal: Create a fun game using the IGIS system using the Swift coding language.

Language: Swift

Concept:

We wanted to base a graphic game on a popular game we played when we were kids called Super Mario Bros. We wanted to include the Mario character, Coins, Question Block Tiles, multiple levels, and we added lives and enemies.

Motivation:

We chose this game because we wanted a challenge. We wanted something that would test our skills.

FRC Scouting System

Goal: Create an android and desktop application that is able to be used to easier gather and report scouting data to our robotics team.

Languages: C++ & Kotlin

Concept:

Wanted to make collecting information on teams easier in order to be able to do more strategy. Our previous system was using paper and pencil. While this did work, it took a lot of effort that gave little information. The app can easily be updated and gives all of the information in an electronic format.

Motivation:

I wanted to speed up our strategy process. It helps a lot to be able to search match by match, by team, by highest average points, etc.

Conclusion:

The original project was in React Native. While this did work, it was messy and I did not like the compiling process. I have now switched to Kotlin, which is specifically for Android devices. This works for us because we have a bunch of old First Tech Challenge (FTC) phones we can scout on. The desktop application previously used electron and react, but the new application uses QT with c++.

3D Video Game

Language: Unreal Blueprint/C++

Concept:

I wanted to create a game that was inspired by CrazyTaxi. I always loved playing that game, and felt it could be made even better with modern graphics and sort other game elements.

Motivation:

I was required to pick a topic for my Independent Study Project that I was interested in. I had never created a 3D game before, so I set that as my goal. Progress:

So far I am about 30 hours in out of the 60 requirement, and I am pretty far along. I have constructed most of the logic behind the game, and what is left is the asset modeling, the animation, the sound, and the tutorials.
